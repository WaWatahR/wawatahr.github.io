D3 Shiny Hello World

Posted by Warren on 30 Jan, 2015

As many data scientists before me I have had to spend some time reading through a number of blogs, Hello World programs and tutorials. I wanted to dumb it down completely that if someone has ever created a Shiny app in RStudio and know the difference between Java nd Javascript that they sould get D3 going in R in just a few moments.

All the code is available on my github account. I suggest downloading it as a good baseline to start any D3 Shiny project. Just update the css as the styling is not ideal.


Let's get going!

The first thing to remember is we are simply preparing and processing the data in R. D3 is doing all the heavy lifting in terms of the javascript code required to produce high touch reporting. For further details see d3js.org or to dive deeper into tutorials check out these.

The first thing we need to understand is that Shiny is a framework that will render HTML and javascript from the code the program is required to be interpreted. In order for the program to incorporate the D3 functions the javascript functions and libaries need to be referenced at page load. 

tags$script(src="https://d3js.org/d3.v3.min.js")

Once this has been referenced we can then make a call to our javascript functions in order for Shiny to call it at runtime. To reference the D3 script we reference it like any other javascript file used in html.

tags$script(src="d3script.js")

In order for the D3 to bind to an HTML element create divs with specific IDs in order to make it easier to manage and control. This is not the only way of doing it just by far the easiet for me.

tags$div(id="d3out"),
                            tags$div(id="d3chart", class="chart")

Now that the UI piece is sorted out we need to setup the d3script javascript and the server file. In order for the R to communicate the data between D3 and Shiny we create a custom message handler. Essentially the server will send the data frame as message which will be handled client side. It is important to note here that D3 expects to receive a JSON message format. Luckily there is the jsonlite that will convert a structured table or list to a JSON array. In order to make the output responsive the data manipulation is wrapped in a reactive function that is referenced in one of the standard "render" functions of Shiny.

d3data <- reactive({
							var_json = toJSON(data.frame(ID= c('Bar 1','Bar 2','Bar 3','Bar 4','Bar 5'), value= c(input$b1,input$b2,input$b3,input$b4,input$b5)))
							session$sendCustomMessage(type="jsondata",var_json})
